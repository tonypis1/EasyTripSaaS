Logica Bubble — Pulsante "Rigenera Itinerario"
Workflow che scatta quando l'utente clicca "Non mi piace, rigenerami"
// Workflow: Button "Rigenera" clicked STEP 1 — Controlla regen_count del Trip corrente IF Trip.regen_count < 3 → gratis GOTO STEP 3 ELSE IF Trip.regen_count >= 3 AND Trip.regen_count < 7 → a pagamento Show popup "Rigenerazione extra: €1,99. Confermi?" IF user confirm: Stripe.charge(€1.99, user.stripe_id) GOTO STEP 3 ELSE: annulla ELSE IF Trip.regen_count = 7 → limite raggiunto: carosello Show popup "Hai raggiunto il limite. Riparti dalla versione 1?" IF user confirm: Set Trip.current_version = 1 Set TripVersion[1].is_active = true Set altri TripVersion.is_active = false END // non chiama Claude, usa versione salvata STEP 2 — (solo versioni 4–7) Addebita €1,99 via Stripe STEP 3 — Genera nuovo itinerario API Call → Claude con prompt arricchito: "Genera un itinerario DIVERSO dal precedente. Ottimizza i punti di interesse per distanza geografica. Destinazione: [dest], Stile: [stile], Giorni: [n] VERSIONE: [regen_count+1] — sii più creativo delle versioni precedenti." STEP 4 — Salva risultato Create TripVersion (version_num = regen_count+1, is_active=true) Create Day records (uno per giorno, con unlock_date calcolato) Update Trip.regen_count += 1 Update Trip.current_version = regen_count Set vecchia TripVersion.is_active = false STEP 5 — Feedback all'utente Show "Itinerario v[N] pronto! 🎉" // Se regen_count >= 2: mostra "Punteggio ottimizzazione distanze: ★★★★☆" // Questo educa l'utente sul geo_score e crea aspettativa
💡 Nota sul geo_score: nel prompt Claude chiedi esplicitamente di ordinare le attività per vicinanza geografica e restituire un "optimization_score" da 1–10. Salvalo nel campo geo_score di TripVersion. Mostralo all'utente come "Efficienza percorso ★★★★☆" — così capisce che la versione 4+ è oggettivamente migliore e smette di rigenerare.

Formula split automatico in Bubble
Workflow che si aggiorna ogni volta che viene aggiunta una spesa
// Quando viene creata una nuova Expense: STEP 1 — Calcola quota per membro quota = Expense.amount / GroupMember:count(Trip = this.trip) STEP 2 — Aggiorna balance di ogni membro FOR EACH GroupMember in this.trip: IF member = Expense.paid_by: member.balance += (Expense.amount - quota) // ha pagato più della sua quota member.total_paid += Expense.amount ELSE: member.balance -= quota // deve la sua quota // Il balance mostra: positivo = deve ricevere, negativo = deve dare // La UI mostra sempre la situazione aggiornata in tempo reale // Nessun calcolo manuale — Bubble lo fa in automatico
04 — Sistema Crediti
Cancellazione → Credito, mai rimborso

Il credito è la scelta più intelligente: trattieni il ricavo, fidelizzi il cliente, e crei un incentivo naturale a tornare. Ecco come costruirlo.

✅
Cancellazione ammessa
L'utente può cancellare il viaggio in qualsiasi momento prima della data di inizio. Dopo l'inizio del viaggio, il trip è considerato "in uso" e non cancellabile.
Regola: solo pre-viaggio
🪙
Credito emesso automaticamente
Bubble crea un record Credit con l'importo pagato. Make invia un'email: "Il tuo credito di €14,99 è pronto per il prossimo viaggio".
Scadenza: 12 mesi
📅
Credito scade
Se non usato entro 12 mesi, il credito scade. Email automatica a 30 giorni, 7 giorni e 1 giorno dalla scadenza. Crea urgenza positiva al riacquisto.
Urgenza = conversione
🛒
Uso del credito
Al checkout del trip successivo, Bubble controlla se l'utente ha Credit.used=false e Credit.expires_at > oggi. Se sì, applica lo sconto automaticamente.
Zero attriti per l'utente
🔄
Workflow cancellazione completo
Da Bubble + Make + Brevo
// Pulsante "Cancella viaggio" premuto dall'organizzatore STEP 1 — Verifica eligibilità IF today >= Trip.start_date: Show "Il viaggio è già iniziato e non può essere cancellato." STOP ELSE: Show confirm dialog: "Cancellando riceverai €[amount] di credito valido 12 mesi. Confermi?" STEP 2 — Cancella il trip Set Trip.status = "cancelled" Set Trip.access_expires_at = now() // accesso immediato revocato // Se gruppo: rimuovi anche l'accesso ai membri STEP 3 — Emetti credito Create Credit: user = Trip.organizer amount = Trip.amount_paid origin_trip = this.trip expires_at = now() + 365 days used = false Update User.credit_balance += Trip.amount_paid STEP 4 — Notifica (via Make → Brevo) Email: "Il tuo credito di €[amount] è attivo! Usalo entro [expires_at]." // Schedule email reminder: -30gg, -7gg, -1gg da expires_at STEP 5 — Uso al prossimo checkout IF User.credit_balance > 0 AND credit.expires_at > today: Apply credito al totale automaticamente Set Credit.used = true Set Credit.used_on_trip = new_trip Update User.credit_balance -= credit.amount
05 — Supporto Clienti
Chat in-app + email: la combo giusta

Il supporto è anche un canale di vendita. Una risposta rapida durante il viaggio vale quanto una buona recensione. Ecco come costruirlo con zero codice.

💬
Chat in-app
Disponibile solo mentre il trip è attivo
→Widget chat visibile nell'angolo in basso a destra durante il viaggio
→Risposta umana entro 4 ore (nelle prime fasi, rispondi tu manualmente)
→Al click "Chat" → apre una conversazione su Crisp o Tidio direttamente nell'app
→Ogni messaggio viene collegato automaticamente all'utente e al suo trip (contesto visibile a te)
→Dopo la scadenza del trip, la chat è visibile in sola lettura ma non inviare nuovi messaggi
🛠 Tool: Crisp.chat
Si integra con Bubble in 5 minuti via script embed. Gratuito fino a 2 agenti.
€0/mese (free tier) → €25/mese (pro)

06 — Workflow Bubble Principali
I 7 automatismi da costruire

Bubble chiama "workflows" le automazioni. Eccoli tutti, con trigger preciso e tool coinvolto.

1
Acquisto completato → Crea utente + Trip
Trigger: Stripe webhook "payment_intent.succeeded"
Make
→Make riceve il webhook Stripe con email, importo, metadata (tipo trip, date)
→Crea User in Bubble (se non esiste) con email + stripe_customer_id
→Crea Trip con tutti i campi (status=active, regen_count=0, access_expires_at = end_date+1)
→Triggera API call Claude per generare itinerario (vedi workflow 2)
→Invia email via Brevo con magic link di accesso
2
Genera itinerario AI → Salva Days
Trigger: Trip creato (o pulsante "Rigenera")
Bubble
→Bubble API Connector chiama Claude con prompt strutturato (destinazione, stile, giorni, versione)
→Claude risponde con JSON: array di giorni con attività mattina/pomeriggio/sera, ristoranti, tips, geo_score
→Bubble crea 1 record TripVersion + N record Day (uno per giorno, unlock_date = start_date + day_number - 1)
→Aggiorna Trip.regen_count, Trip.current_version, set vecchia versione is_active=false
3
Sblocco giornaliero + notifica push
Trigger: Scheduled API workflow ogni giorno alle 06:00 UTC+1
Bubble
→Bubble scheduled workflow: cerca tutti i Day dove unlock_date = oggi e trip.status = active
→Invia notifica push via OneSignal: "Buongiorno da [città]! Il tuo programma per oggi è pronto ☀️"
→La UI mostra automaticamente il day corrente in highlight (Bubble: condizione "Day.unlock_date = today")
4
Scadenza accesso + schermata post-trip
Trigger: Scheduled workflow giornaliero + condizione pagina
Bubble
→Ogni pagina trip: IF today > Trip.access_expires_at → mostra schermata "Viaggio terminato" invece del contenuto
→Scheduled workflow: cambia Trip.status = "expired" a mezzanotte di access_expires_at
→Mostra CTA: "Riattiva per €2,90 (30 giorni)" e "Pianifica il prossimo viaggio"
→Invia email automatica (Brevo): "Il tuo viaggio è terminato. Come riattivare →"
5
Riattivazione post-trip (€2,90)
Trigger: Click pulsante "Riattiva" sulla schermata scaduta
Stripe
→Stripe Payment Intent per €2,90 (one-time) — carta già salvata se abbonato
→Se pagamento ok: aggiorna Trip.access_expires_at = today + 30 giorni, Trip.status = "reactivated"
→Sblocca accesso immediato a tutti i Day del trip (in sola lettura — no più unlock progressivo)
→Invia email: "Accesso riattivato! Rivivi il tuo viaggio a [città] 🌟"
6
Cancellazione trip → Emissione credito
Trigger: Click "Cancella viaggio" (solo organizzatore, solo pre-inizio)
Bubble
→Verifica: IF today >= Trip.start_date → mostra errore "Non cancellabile dopo l'inizio"
→Crea record Credit (amount=Trip.amount_paid, expires_at=now+365d, used=false)
→Aggiorna Trip.status="cancelled", rimuovi accesso a tutti i GroupMember
→Aggiorna User.credit_balance += amount
→Email Brevo: "Credito €[X] attivo — usalo entro [data]. Non scade prima!" (tono positivo)
7
Aggiunta spesa gruppo → Ricalcolo split
Trigger: Creazione nuova Expense da qualsiasi membro del gruppo
Bubble
→Calcola quota: Expense.amount ÷ numero GroupMember attivi nel trip
→Loop su ogni GroupMember: aggiorna balance (+ o − quota a seconda di chi ha pagato)
→Aggiorna GroupMember.total_paid del pagante
→La tabella riepilogo si aggiorna in real-time per tutti i membri (Bubble: live data binding)
07 — Ordine di Costruzione
Cosa costruire prima, cosa dopo

Non costruire tutto insieme. Inizia dal cuore del prodotto e aggiungi complessità solo dopo aver validato ogni livello con utenti reali.

Priorità Feature Perché prima Complessità Bubble Settimana target
P1 — Must Pagamento Stripe → crea Trip Senza questo, niente prodotto Media Sett. 1–2
P1 — Must Generazione itinerario Claude API È il core value — senza non esiste l'app Media Sett. 3–4
P1 — Must Sblocco giornaliero + scadenza accesso È il modello di business — senza non si monetizza Bassa Sett. 4–5
P1 — Must Email onboarding automatiche Senza email il 60% degli utenti non rientra Bassa Sett. 5
P2 — Should Rigenerazione itinerario (3 gratis + carosello) Alta retention + revenue extra Media Sett. 6
P2 — Should Sistema crediti cancellazione Riduce churn, evita dispute Stripe Bassa Sett. 6–7
P2 — Should Link invito gruppo + accesso membri Sblocca il tier €14,99 (+ 50% revenue) Media Sett. 7
P2 — Should Chat supporto (Crisp embed) Riduce refund, aumenta soddisfazione Bassa Sett. 7–8
P3 — Nice Split spese gruppo (Expense table) Valore aggiunto, non critico al lancio Media Sett. 8–9
P3 — Nice LocalPass layer (gemme locali) Revenue add-on, costruisce su base solida Bassa Sett. 9
P3 — Nice Riattivazione post-trip €2,90 Revenue passivo, aggiungibile in 2 ore Bassa Sett. 9–10
P3 — Nice Notifiche push (OneSignal) Engagement booster, non bloccante Bassa Post-lancio
Il prossimo step concreto
  • 1
    Oggi: Apri Bubble → crea una nuova app vuota → crea le 8 tabelle del database esattamente come descritte in questo documento. Ci vuole 1–2 ore.
  • 2
    Questa settimana: Configura Stripe in test mode con 2 prodotti: "Trip Solo €9,99" e "Trip Gruppo €14,99". Crea il flow Make: Stripe → Bubble → Brevo.
  • 3
    Settimana 2: Scrivi il prompt Claude master per la generazione itinerari. Testalo direttamente su claude.ai fino a quando il JSON output è esattamente quello che vuoi. Solo dopo integralo in Bubble.
  • 4
    Prima del lancio: Fai almeno 3 "viaggi test" completi tu stesso: compra, genera, simula lo sblocco giornaliero, cancella, usa il credito. Solo così trovi i bug prima degli utenti.
Una cosa da non dimenticare mai

Il prodotto che stai costruendo non è un'app. È un rituale di viaggio. Ogni volta che un utente apre EasyTrip la mattina del viaggio, stai competendo con l'ansia del "cosa facciamo oggi?". Vinci quella gara e hai un prodotto che le persone amano.
