Specifiche tecniche — modello dati e API (Next.js)

Schema Prisma: Trip, TripVersion (con geo_score), Day (slot JSON, ristoranti, centro mappa, zone, avvisi).

Campi principali Day

CampoRuolo
morning / afternoon / eveningJSON slot (titolo, luogo, perché, orari, tips)
restaurantsJSON array suggerimenti ristoro
map_center_lat / lngCentro giornata per link mappa
zone_focusZona per diversificare rigenerazioni (used_zones sul Trip)
dow_warningAvvisi giorno settimana / chiusure
local_gemSuggerimento da locale
tipsNote trasversali al giorno

Endpoint rilevanti

  • POST /api/billing/checkout — acquisto viaggio
  • POST /api/billing/regen-checkout — rigenerazione a pagamento
  • POST /api/webhooks/stripe — pagamenti e avvio generazione
  • POST /api/trips/:tripId/generate — generazione gratuita / prima generazione (in dev anche senza pagamento)
  • POST /api/trips/:tripId/active-version — carosello
  • POST /api/trips/:tripId/replace-slot — sostituzione slot con AI
