Uso realeProduzione — core feature
Perché la differenza di 44m conta: In un quartiere denso come Poble Sec, 44m di errore possono significare suggerire una strada diversa come "più vicina". Con GPS reale le distanze sono precise al metro — Claude può dire "gira a sinistra, 3 minuti" invece di "circa 5 minuti nella direzione generale". È la differenza tra un'app utile e una che sembra indovinare.
Gestione edge cases GPS
⚠️
GPS non disponibile
Utente nega il permesso o è in zona coperta. Bubble usa l'indirizzo dell'hotel (salvato al checkout) come fallback. Accuracy = ~500m.
📡
Accuracy > 100m
Se il GPS ha accuracy bassa, aggiungi al prompt: "GPS accuracy: {{accuracy}}m — allarga il raggio di suggerimento a 1.5km". Claude si adatta.
🗺️
Utente fuori dalla città
Se lat/lng sono > 20km dalla destinazione del Trip, mostra alert "Sembra che tu sia fuori da Barcellona — i suggerimenti si basano sul tuo piano originale."
🔋
Risparmio batteria
Non fare geolocation continua. Richiedila solo quando l'utente clicca "Cosa faccio adesso?" — una singola call GPS, poi passa le coords a Claude.
Implementazione Bubble
Il bottone "Cosa faccio adesso?"
passo per passo.

Come costruire in Bubble il workflow completo che prende il GPS, chiama Claude e mostra i 3 suggerimenti — tutto senza scrivere codice.

1
Abilita il plugin Geolocation in Bubble
Plugins → Add → "Browser GPS" (gratuito)
Facile
→Menu laterale → Plugins → "Add plugins" → cerca "Browser GPS" → Install (gratuito)
→Nella pagina del viaggio: Settings → Geography → abilita "Expose geographic location"
→Questo rende disponibili automaticamente le variabili Current Geographic Position latitude e longitude in tutta la pagina
→Bubble chiederà il permesso all'utente alla prima visita — gestisce tutto lui, zero codice
2
Aggiungi il bottone "Cosa faccio adesso?"
Design → Visual elements → Button
Facile
→Nella pagina del giorno corrente, aggiungi un Button → testo: "📍 Cosa faccio adesso?"
→Posizionalo in basso nella schermata, sempre visibile (usa una "floating group" che rimane fissa in fondo)
→Aggiungi un secondo bottone (piccolo) "Il posto era chiuso" e uno "Troppo affollato" — passano la reason_for_change diversa allo stesso workflow
3
Workflow: "When Button clicked → Claude API"
Workflow editor → API call → parse response
Media
→Workflow → "When Cosa faccio adesso button is clicked"
→Step 1: Show Popup "LiveSuggestionsLoading" (popup con spinner — mostra subito, l'utente sa che sta elaborando)
→Step 2: API Call → ClaudeAPI → LiveSuggestions (la chiamata API che hai configurato nell'API Connector)
→Nel body dell'API call, inietta: Current Geographic Position latitude e longitude come valori dei campi lat/lng
→Step 3: Hide Popup loading → Show Popup "LiveSuggestions" con i risultati parsati
4
Body API — esatto da incollare nell'API Connector
ClaudeAPI → New call: LiveSuggestions
Copia-incolla
Body JSON — LiveSuggestions callBUBBLE API CONNECTOR
{
  "model": "claude-sonnet-4-20250514",
  "max_tokens": 1000,
  "system": "Sei EasyTrip AI in modalità live. Rispondi SOLO in JSON valido. Max 3 suggerimenti. Orario locale: Europe/Madrid.",
  "messages": [{
    "role": "user",
    "content": "{\"current_position\":{\"lat\":<gps_lat>,\"lng\":<gps_lng>,\"accuracy\":<gps_accuracy>},\"current_time_local\":\"<local_time>\",\"current_day_number\":<day_number>,\"original_planned_activity\":\"<activity_name>\",\"reason_for_change\":\"<reason>\",\"remaining_time_minutes\":<minutes_left>,\"travel_style\":\"<style>\"}"
  }]
}

// I valori tra < > sono variabili Bubble:
// <gps_lat> = Current Geographic Position latitude
// <gps_lng> = Current Geographic Position longitude
// <gps_accuracy> = Current Geographic Position accuracy
// <local_time> = Current date/time :formatted as "HH:mm"
// <day_number> = Current Trip's day_number (calcolato da start_date)
// <activity_name> = Current Day's planned activity name
// <reason> = URL param o stato del bottone cliccato
// <minutes_left> = differenza tra ora attuale e prossimo slot programmato
5
Mostrare i risultati nella UI
Popup con 3 card ripetute dinamicamente
Facile
→La risposta Claude è un JSON — Bubble la legge come Result of step 2's body
→Usa l'operatore ":extract with regex" o "detect data type" di Bubble per parsare il JSON della risposta
→In alternativa più semplice: salva la risposta grezza in una custom state → usa un Repeating Group con 3 righe hardcoded che leggono suggestions[0], suggestions[1], suggestions[2]
→Ogni card mostra: nome, distanza in minuti, insider_tip e un link "Apri in Maps" con https://maps.google.com?q=lat,lng
→📨 Cosa dirmi a questo punto: "Ho configurato il workflow ma quando leggo la risposta Claude con :extract non trovo il campo suggestions. Come faccio?" — Ti guido campo per campo.
