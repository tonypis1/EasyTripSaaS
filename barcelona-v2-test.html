System Prompt — Versione Aggiornata
Prompt con day-of-week logic integrata

La nuova sezione da aggiungere al prompt esistente + i nuovi parametri nel user message. Le righe evidenziate in verde sono le aggiunte rispetto alla versione precedente.

Sezione da aggiungere — dopo "AUTENTICITÀ"
System Prompt — Nuova sezioneAGGIUNTA
LOGICA GIORNO DELLA SETTIMANA (obbligatorio):
— Ricevi start_date e il giorno della settimana per ogni giorno del trip
— Per ogni attività, ristorante o mercato che ha restrizioni di apertura:
  • Aggiungi il campo "day_of_week_note" con la restrizione SPECIFICA
  • Aggiungi SEMPRE un'alternativa concreta entro 500m per i giorni di chiusura
  • Se il posto è aperto senza restrizioni: "day_of_week_note": null

— Regole Barcellona da applicare sempre:
  • Lunedì: molti ristoranti indipendenti chiusi — verifica e dai alternativa
  • Domenica mattina: mercati coperti chiusi, usa solo banchi esterni
  • Domenica sera: ristoranti haute cuisine spesso chiusi
  • Sabato/domenica: aste del pesce non operative la domenica
  • Primo weekend del mese: mercati artigianali aperti (Palo Alto, etc.)
  • Agosto (1–31): fino al 40% dei ristoranti locali chiusi per ferie
    → in agosto privilegia posti con staff fisso o catene artigianali

— Aggiungi al JSON root il campo "day_of_week_warnings" con:
  • Lista di avvisi specifici per il periodo del viaggio
  • Campo "note" che spiega la logica

Bubble — Day of Week Configuration
Come calcolare i giorni della settimana in Bubble

Bubble ha operatori nativi per formattare le date. Zero plugin, zero codice. Ecco la configurazione esatta campo per campo.

1
Aggiungi campo "used_zones" alla tabella Trip
Data → Trip → Create a new field
Facile
→Data → Trip → Create a new field → Nome: used_zones → Tipo: text → Save
→Questo campo accumula i quartieri usati ad ogni rigenera: "El Born, Gràcia, Poble Sec"
→Dopo ogni generazione: workflow Step → Make changes to Trip → used_zones = usato_prima + ", " + nuovo_zone_focus
2
Calcolare il giorno della settimana da start_date
Nessun plugin — usa l'operatore :formatted as
Facile
→Nel workflow di generazione itinerario, quando costruisci il body della API call:
→Per il Giorno 1: usa Trip's start_date:formatted as "dddd" → restituisce "Friday"
→Per il Giorno 2: Trip's start_date + 1 day:formatted as "dddd" → "Saturday"
→Per il Giorno N: Trip's start_date + N-1 days:formatted as "dddd"
→Bubble restituisce l'inglese → traducilo in italiano nella stringa del prompt: sostituisci "Friday" con "venerdì" ecc.
3
Calcolare la stagione da start_date
Conditional expression in Bubble
Facile
→Nel body JSON dell'API call, per il campo "season" usa questa logica Bubble:
→Se start_date:month is 12 or 1 or 2 → "inverno"
→Se start_date:month is 3 or 4 or 5 → "primavera"
→Se start_date:month is 6 or 7 or 8 → "estate"
→Se start_date:month is 9 or 10 or 11 → "autunno"
→In Bubble usa un "Conditional" su un campo Text invisibile nella pagina, oppure usa l'operatore :is direttamente nel testo del body JSON
4
Leggere day_of_week_note dal JSON e mostrarlo in app
Bubble JSON extraction + UI
Facile
→Claude restituisce day_of_week_note per ogni attività nel JSON
→Quando crei i record Day in Bubble, aggiungi campo dow_warning (tipo: text) e salvaci questo valore
→Nella UI: aggiungi un elemento Text con condizione Day's dow_warning is not empty → mostra il testo con sfondo ambra
→Apparirà automaticamente solo per le attività che hanno restrizioni — zero noise per quelle senza problemi
